1 #psl-apiclient
2 Biblioteca .NET que permite acceder a los servicios de la API Pasamonte Labs.
3 Encapsula invocaciones REST a través de una biblioteca de clases, facilitando realizar la integración con los servicios.
4 El servicio REST también es accesible a través de HTTP. En la parte inferior de esta descripción se presentan ejemplos de invocaciones utilizando el utilitario de comandos *curl*.
5 
6 ##Los servicios disponibles se agrupan en dos categorías:
7 
8 ###1. Integración con Registro Clínico Electrónico (RCE). (Actualmente se implementa la interoperabilidad con la plataforma SAYDEX).
9 * Validación de usuarios (pacientes) en plataforma RCE.
10 * Obtención de recetas con despachos pendientes desde RCE.
11 * Registro (notificación) de despachos realizados a RCE.
12 
13 ###2. Integración con operaciones de
14 * Citas médicas
15 * Administración de turnos (Servicios, Módulos, Turnos)
16

39 ##Uso de los métodos desde línea de comandos
40 Se puede acceder a los métodos utilizando *curl* desde la línea de comandos, siguiendo los parámetros indicados en la documentación de cada método.
41 En cada llamada se debe proveer el parámetro **apiKey** y debe invocarse utilizando el verbo **POST**.
42 
43 **Ejemplo:**
44 * Para obtener una lista de los terminales registrados en el sistema, un comando posible es:
45 ```
46 curl -H "Content-Type: application/json" -i \
47  -X POST http://ppi.cloudapp.net/api/adm/obtenerterminales \
48  -d "{ 'apiKey': 'a34287f8-102a-4cf3-bace-153c30c3eb58', \
49  'idTerminal': 'a34287f8-102a-4cf3-bace-153c30c3eb58' }"
50 ```
51 * Para obtener los datos de un terminal específico, se agrega el id del terminal como parámetro:
52 ```
53 curl -H "Content-Type: application/json" -i \
54  -X POST http://ppi.cloudapp.net/api/adm/obtenerterminales \
55  -d "{ 'apiKey': 'a34287f8-102a-4cf3-bace-153c30c3eb58', \
56  'idTerminal': 'a34287f8-102a-4cf3-bace-153c30c3eb58', \
57  'id': '7a873d2f-181d-4c23-90c4-176897f47382' }"
58 ```
59 * Para crear un servicio, debemos proveer los siguientes parámetros:
60 ```
61 +-----------------------------------------------------------------------------------------------------+
62 | parámetro | opcional | valor por defecto | ejemplo |
63 +-----------------------------------------------------------------------------------------------------+
64 | id | sí | <nulo> | 75e48afb-25dd-447a-a864-15992ec070b4 |
65 +-----------------------------------------------------------------------------------------------------+
66 | titulo | no | | servicio de prueba |
67 +-----------------------------------------------------------------------------------------------------+
68 | activo | no | | true |
69 +-----------------------------------------------------------------------------------------------------+
70 | bloqueado | no | | false |
71 +-----------------------------------------------------------------------------------------------------+
72 | idNodo | no | | f36a323c-3c20-42eb-811c-4e84668adade |
73 +-----------------------------------------------------------------------------------------------------+
74 | configuracionJson | sí | <nulo> | |
75 +-----------------------------------------------------------------------------------------------------+
76 | tipoDeToken | no | | numerico |
77 +-----------------------------------------------------------------------------------------------------+
78 | tokenMaximoCorrelativo | no | | 100 |
79 +-----------------------------------------------------------------------------------------------------+
80 | ultimoTokenGenerado | sí | <nulo> | 1 |
81 +-----------------------------------------------------------------------------------------------------+
82 | prioridadVisual | no | | alta |
83 +-----------------------------------------------------------------------------------------------------+
84 ```
85 el comando resultante sería:
86 ```
87 curl -H "Content-Type: application/json" -i \
88  -X POST http://ppi.cloudapp.net/api/adm/registrarservicio \
89  -d "{ 'apiKey': 'a34287f8-102a-4cf3-bace-153c30c3eb58', \
90  'id': '75e48afb-25dd-447a-a864-15992ec070b4', \
91  'titulo': 'servicio de prueba', \
92  'activo': 'true', \
93  'bloqueado': 'false', \
94  'idNodo': 'f36a323c-3c20-42eb-811c-4e84668adade', \
95  'configuracionJson': '', \
96  'tipoDeToken': 'numerico', \
97  'tokenMaximoCorrelativo': '100', \
98  'ultimoTokenGenerado': 1, \
99  'prioridadVisual': 'alta' }"
100 ```
